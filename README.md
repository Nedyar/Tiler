# Tiler

Tiler is a project composed of two independent modules:

- **Generator** (Java)
- **Visualizer** (JavaScript)

Together, they allow generating all valid tile boards of size **N x M**
and visualizing them using sprites.

# ðŸ”· Generator (Java)

The **Generator** module is a Java program responsible for:

- Generating all valid boards of size **N x M**
- Using predefined tiles configured in the `Main` class
- Ensuring no duplicated boards are stored under full board rotation
- Saving the generated boards into JSON files
- Storing output files inside the `logs` directory

---

## âœ… Rotation Deduplication

The Generator guarantees that:

> Boards that are identical under complete board rotation are not stored
> multiple times.

This prevents redundant configurations and significantly reduces output
size.

---

## ðŸ“¦ Output Size

Generated JSON files may be very large:

- Approximately **1GB total output** for boards greater than 4x4
- Automatically split into multiple files if necessary

This ensures the system can handle very large board spaces without
failing due to file size limits.

---

## ðŸ§© Preconfigured Tile Setups

The Generator includes two predefined tile configuration methods:

- `setUpAmonHen`
- `setUpAmonSun`

These methods:

- Predefine specific tile configurations
- Serve as examples of how to generate boards with pre-established
  tile sets
- Can be used as references for creating custom predefined tile
  configurations

---

## ðŸ”„ How the Generator Works

1.  Tiles are configured in `Main`.
2.  Optionally use `setUpAmonHen` or `setUpAmonSun` for predefined
    setups.
3.  The program computes all valid board configurations.
4.  Duplicate boards under full rotation are discarded.
5.  Results are exported into JSON files.
6.  Files are saved inside the `logs` folder.

---

# ðŸ”· Visualizer (JavaScript)

The **Visualizer** module is a JavaScript program responsible for:

- Reading a JSON file generated by the Generator
- Selecting a random valid board
- Rendering the board visually
- Using sprite images stored in the `sprites` folder

## ðŸš€ Workflow

1.  Run the **Generator**.
2.  Generate valid boards (may produce multiple large JSON files in
    `logs/`).
3.  Prepare the desired JSON file:
    - Rename or merge into `validBoards.json`
